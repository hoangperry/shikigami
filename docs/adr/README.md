# Architecture Decision Records

> We use lightweight ADRs to capture architectural decisions with full context and trade-offs. Each ADR is immutable once **Accepted** — to change a decision, write a new ADR that supersedes the old one.

## Template

```markdown
# ADR-XXX: <Title>

**Status**: Proposed | Accepted | Deprecated | Superseded by ADR-YYY
**Date**: YYYY-MM-DD
**Deciders**: <names>

## Context
<What problem are we solving? What forces are at play?>

## Decision
<What we decided, in clear terms.>

## Consequences
<Positive, negative, neutral — all trade-offs explicit.>

## Alternatives Considered
<What else did we look at? Why rejected?>

## References
<Links, docs, reviews that informed this decision.>
```
